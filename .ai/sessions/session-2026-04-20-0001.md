### 2.2. v0.1.4 신규 모듈 코드 리뷰 (skills / langgraph / hitl.interrupt)

사용자 요청으로 주말(2026-04-19) 작업분 v0.1.4 의 신규 모듈 3개를
실제 코드와 함께 정리:

- **`minyoung_mah.skills`** — `Skill` frozen dataclass (`name`,
  `summary`, `applies_to`, `body`, `path`), `SkillStore(root)` 가 `*.md`
  를 eager load 하고 `_by_name`/`_by_role` 두 인덱스 유지,
  `parse_frontmatter` 는 pyyaml 의존성 없는 미니멀 파서,
  `render_skill_block(skills)` 가 user message 용 markdown 블록 렌더.
  설계 원칙: role system prompt = 정체성 계약, SKILL.md = how-to 절차.
  module-level 싱글톤 미노출 — root path 는 컨슈머 결정.
- **`minyoung_mah.hitl.interrupt`** — `HITL_INTERRUPT_MARKER = "__mm_interrupt__"`
  상수화 (값 rename 으로 기존 in-flight checkpoint 무효화),
  `make_interrupt_marker(payload)` / `extract_interrupt_payload(value)`
  헬퍼. SubAgent 의 `ask` 도구가 LangGraph `interrupt()` 를 직접 호출하면
  라이브러리 tool-loop 를 unwind 하므로, 도구는 마커 dict 만 반환하고
  outer driver 가 마커 발견 시 `interrupt()` 호출.
- **`minyoung_mah.langgraph`** (optional extra `pip install minyoung-mah[langgraph]`,
  `langgraph>=0.4`) — `build_subagent_task_tool(orchestrator, ...)` 가
  replay-safe LangGraph `task` 도구 빌더. `replay_safe_tool_call(tool_call_id)`
  컨텍스트 매니저가 `GraphInterrupt` 에만 cache 보존. 핵심 함정: LangGraph
  `interrupt()` 는 resume 시 도구 함수 전체를 replay 하여 SubAgent 호출이
  round 1만큼 drift → `_TOOL_CALL_CACHE[(tool_call_id, iter_idx)]` 에
  `RoleInvocationResult` memoize 로 cache hit 시 invoke skip + `interrupt()`
  가 저장된 답변 즉시 반환. 6개 hook (`resolve_role`, `format_result`,
  `format_failure`, `format_hitl_answer`, `on_tool_call_start`,
  `on_tool_call_end`, `on_user_answer`) 노출 — 라이브러리는 replay-safety
  루프만 소유, role resolution / todo ledger / 결과 렌더링은 컨슈머.
- 부수 변경: observer event rename `orchestrator.tool.*` → `role.tool.*`
  (실제 발화 위치가 role invocation 내부 `ToolInvocationEngine` 이라
  namespace 정정). 기존 대시보드는 키 갱신 필요.
